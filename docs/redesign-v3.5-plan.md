# 重构方向 v3.5（基于多智能体调研）

调研结论（竞品 + 动线诊断两份报告）的核心主线：
**从"展示数据的面板"改成"给一个能照做的结论的决策器"**。

## 本次重构范围（按优先级）

### 1. 时间换算提升为一等公民（用户明确想要）
- 现状：藏在 qd 页底部折叠 `qdPlanFold`，`advanced-only`，简化版完全看不到，标题叫"时间换算"（搜不到用户意图）。
- 改造：做成 Wise 式双向计算器 —— 两个方向（"几点取号→几点吃" / "想几点吃→几点取号"）用 **⇄ 换向按钮**原地翻转，输入即算（去掉"算一下"按钮），结果卡片三件套（主结果大字 + 区间 + 可展开依据）。
- 位置：qd 页首屏可见卡片（不再折叠、不再 advanced-only），并简化版可见。
- 倒推方向加风险提示"取号后实际等待可能 ±15 分钟"。

### 2. 设置页重构（用户明确说"操作很迷"）
- 现状：7 个折叠全关、危险操作和配置混排、查看类（日志/洞察）塞进设置、核心业务偏好（人数/门店）反而在 qd 页。
- 改造（NN/G 范式）：
  - 分组卡片**直接展开**（不全折叠）：① 认证（通行证 + GitHub）② 通知 ③ 数据/预测（采集 + 洞察）④ 维护（日志 + 安全）
  - 认证相邻、通知单独、查看类（日志/洞察）降级但仍可访问
  - **危险操作独立 Danger Zone**（红框、底部、与良性配置大间距）
  - 通行证单独成块带状态指示
  - 折叠只留给"解释性长文 + 罕见诊断开关"

### 3. 整体动线 + 样式不足
- 导航分组优化：number 单页组并入 eat（合并为"今天/排队"，qt+qd 子导航）
- 决策结论前置：qd 页实时压力卡里"建议几点出发"用主色加粗突出
- 简化版边界修正：时间换算对所有模式可见

## 不做（本次范围外）
- 位置进度条替换图表（P2，工程量大，单独迭代）
- 自适应轮询频率（P2，需后端改动）
- 暗色模式、SSE 心跳（体验增强，非动线问题）

## 执行方式
所有改动集中在一个文件 internal/app/web_static.go。用多智能体并行**设计/草拟**新组件（设置页、时间换算器、CSS），主线集成 + 完整回归测试由我串起来。
