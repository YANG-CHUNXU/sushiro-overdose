# Sushiro Overdose 2.20.0 Release Notes

2.20.0 是一个「让更多人能用上、链路更清晰」的体验版本：统一用词、首次打开自动引导、把 Windows 用手机获取凭证的流程做到普通人能跟着走，首页提醒通知是否配置；并把排队相关页面从「数据监控」重构为「吃饭决策助手」。功能能力和安全边界保持不变。

## 排队体验重构：从看曲线到给答案

- **统一概念「排队压力」**：用户侧只看 低 / 中 / 高 / 极高 / 数据不足，不再出现「最新发出号」等不严谨命名。
- **「我有号码」先给答案**：顶部一句话直接回答「你是 X 号，当前叫到 Y，预计 A-B 分钟后叫到，建议几点出发」，配状态卡片（当前叫到 / 还差 N 号 / 排队压力 / 消化趋势 / 近 15 分钟叫号 / 预计叫到 / 通知状态）；复杂模型曲线、10 分钟叫号表、日期类型拆分收进「高级」展开。
- **新主图**：叫号进度阶梯线 + 排队压力面积 + 我的号 + 现在时间线，tooltip 用人话说明。
- **两个时间互推页**：「取号 → 几点能吃」「想几点吃 → 几点取号」，基于同门店同星期历史等待叠加今日实时压力。
- **「现在去吃」多门店推荐**：按排队压力从低到高排序，告诉你去哪家更可能快点吃上；数据不足的门店明确标注，不臆造。
- 全程只用官方公开接口 + 本机采集 + 历史数据，零外部依赖；自动取号 / 取消号仍是独立的「会执行操作」并保留确认。

## 主要变化

- **全站「认证」统一改为「凭证」**：界面、提示、诊断、错误文案口径一致，更贴近用户日常说法。错误归类逻辑同时兼容旧文案，不影响已保存的状态。
- **首次打开自动引导一次**：还没获取凭证、也没用过时，首页会自动弹出「你现在想做什么」场景选择（只读功能直接进入，需要凭证或会执行操作的功能会先提示）。看过一次后不再打扰。
- **Windows 用手机获取凭证流程重做**：
  - 向导首页用「手机抓一次 → 发到电脑 → 粘贴这里」三步总览讲清整体路径。
  - 手动导入（最稳、不依赖路由器）默认前置；抓包步骤按 iPhone / 安卓分别给出可跟随的清单。
  - 新增「怎么把手机抓到的内容弄到电脑」指引，推荐微信「文件传输助手」，覆盖手机电脑不在同一网络的情况。
  - 解析缺字段时，会提示「还差一次门店/排队请求」或「还差一次我的预约/预约请求」，更好补齐。
  - 自动代理抓页的「改用手动导入」做成可点按钮，遇到 AP 隔离一键退回。
- **通知未配置提醒**：已获取凭证但没配任何通知渠道时，首页会给一条弱提示——抢到预约 / 叫号提醒不会推送，可一键去设置。

## 不包含的变化

- 不新增任何取号、取消预约或取消排队行为，不改动这些操作的确认边界。
- 不改抓包、下单核心逻辑与凭证解析能力，API 路径与字段名保持不变。
- 不引入任何外部依赖（仍为纯 Go 标准库）。
