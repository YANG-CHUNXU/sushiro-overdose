# v3.13.2

## fix: 凭证采集引导动作纠正——真的排队/预约，不是点"我的预约"

用户反馈"抓不全通行证"。根因发现：之前向导写"点我的预约"，但光看「我的预约」列表（GET）
不触发带预约 auth + wechatId/phoneNumber 的 POST。正确动作是**真的排队/预约一下**（触发
createReservation/createNetTicket），抓到后再取消。

### 改动（文案纠正，全 7 处）

- 向导第 2 步：编号 3 从"点我的预约"改为"真的排队或预约一下（之后可取消）"
- SVG 流程图 ②：从"点我的预约"改为"排队或预约一下（之后可取消）"
- 进度提示、PC 启动 toast、第 3 步、自动代理引导、手机引导页、缺字段提示——全部同步纠正
- 加说明："光看「我的预约」列表不行，得真的提交一次排队/预约"

### 为什么

产生"预约请求"（含预约 auth + 微信ID + 手机号）的是提交排队/取号/预约的 POST，
不是查看列表的请求。之前引导让用户"点我的预约"，多半只触发列表查询，抓不到完整的预约类凭证。

> 仅改前端文案（web_static.go + mobile_auth_capture.go），逻辑无改动。
