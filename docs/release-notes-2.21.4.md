# 2.21.4 Release Notes

2.21.4 是排队提醒和到店预测可信度的小修版。

## 修复

- 多段到店提醒再次生成时会替换当前门店和号码的旧提醒点，避免修改路程、标签或模板后仍保留旧规则。
- 叫号提醒触发后再发送通知，避免飞书、Telegram、Bark 等慢响应阻塞提醒配置读写。
- “我有号码”场景必须先选门店，避免自动套用其他门店曲线。
- 只有官方等待数据时不再输出强 ETA，只提示它只能代表门店排队压力。

## 体验

- 生成临时提醒时会尝试把当前门店加入本机持续采集；没有凭证、采集或通知时给出明确提示。
- GitHub 线上基准默认使用 `https://sushiro-cloud.ryujo.online`，普通用户不需要先进入调试模式填写服务地址。
- 取号/就餐倒推文案改为“历史等待为主，实时压力修正风险”，避免过度承诺。
