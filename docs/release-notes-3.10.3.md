# v3.10.3

## feat: 采集器整点对齐调度

采集器改为**整点对齐**采集（:00 :15 :30 :45），不再是"启动即采 + 固定 900s 间隔"。

### 为什么

- 快照时间落在整点边界，与 30min 聚合桶（:00/:30）对齐，数据更规整
- 每小时 4 次、固定时点，便于运维和用户预期（"几点会有新数据"可预测）
- 重启后不会从任意时刻漂移，始终对齐整点

### 改动（`collector/runner.py`）

- 新增 `_seconds_until_next_boundary`：算到下一个整点边界的秒数（interval 能整除 3600 时对齐，否则退化为固定间隔）
- 启动后先睡到下一个 :00/:15/:30/:45 边界，再开始第一轮采集
- 每轮采集后、非营业时段跳过后，都按对齐边界等待下一轮

实测：12:23 启动 → 等 372s → 12:30 开始首轮 → 之后 12:45/13:00/13:15... 对齐整点。

> 仅改 `collector/`（Python），桌面端 Go 代码无改动。
