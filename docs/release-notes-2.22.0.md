# 2.22.0 Release Notes

2.22.0 增加「每日到店 Routine」：用户只需要设置门店、想几点吃、路程和安全提前量，应用会每天按历史等待倒推取号时间，并复用现有自动取号引擎到点执行。

## 新功能

- 「现在去吃」页新增每日 Routine 配置，放在主要区域，不再需要用户手动先去只读倒推页算时间。
- Routine 会把当天推算结果写成一次性自动取号计划，到点后仍走原来的远程取号、凭证过期提示、重复取号恢复和通知流程。
- Routine 生成的计划带 `routine` 来源标记，不会覆盖用户手动设置的自动取号计划。

## 安全边界

- 历史样本不足时只显示 `waiting_data`，不会凭空猜测取号时间，也不会提交取号。
- 关闭 Routine 会清掉还没执行的 routine 计划，但不会取消已经拿到的排队号。
- 当天已经执行、成功、过窗口或手动计划存在时，Routine 会进入对应状态并停止重复执行。

## 验证

- 增加 Routine 状态机测试：按历史样本倒推、保护手动计划、关闭时清理待执行 routine、样本不足不执行。
- 补充本地 API smoke：`/api/queue/ticket/routine` 支持读取、关闭和无样本开启场景。
