# Sushiro Overdose 2.16.0 Release Notes

2.16.0 是一次产品体验重构，把旧的功能导航改成“寿司郎到店助手”任务流。目标是让没用过寿司郎、没理解预约/排队区别的新用户，也能先完成最常见的判断：现在去哪家店、手里的号几点叫到、什么时候需要认证。

## 主要变化

- 首页改成场景入口：现在去吃、我有号码、预约未来、自动蹲号。
- 增加新手向导：用户先选场景，只读功能直接进入，需要认证或会执行操作的功能再提示。
- 增加安全分级：只读、需要认证、会执行操作，减少误点取消预约/排队号的风险。
- “排队”重命名为“现在去吃”，突出实时门店和等位判断。
- “数据看板”重命名为“我有号码”，突出输入号码后估算叫到时间和到店时间。
- 设置页拆出常用设置与高级工具，高级认证、诊断、接口调试不再和普通偏好混在一起。
- 远程取号、自动取号、自动抢预约、自动蹲号、取消预约、取消排队号增加更明确的确认文案。

## 不包含的变化

- 不改官方 API 调用链路。
- 不新增自动取消能力。
- 不修改抢预约核心算法。
- 不把项目改成前后端分离。

## 验收点

- 第一次打开首页，用户可以在 10 秒内看到四个场景入口。
- 用户能区分只读功能和会执行操作的功能。
- 未认证时也能进入“现在去吃”和“我有号码”。
- 输入号码的入口在“我有号码”页首屏可见。
- 危险操作都有明确二次确认。
