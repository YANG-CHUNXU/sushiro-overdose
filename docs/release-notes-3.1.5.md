# Release notes: v3.1.5

3.1.5 聚焦用户体验和线上基准链路：把首页从“功能列表”改成“下一步指挥台”，把诊断页从“状态集合”改成“先处理哪件事”，并修复 GitHub 登录后 Turso 基准没有被真正验证的问题。

## 变化

### 首页

- 新增“今天该走哪条路”面板，根据通行证、运行状态、门店偏好和未完成单据动态给出下一步。
- 面板固定解释“只读 / 通行证 / 会执行”，让用户知道哪些能直接看、哪些需要登录、哪些会真的提交动作。
- 有未完成预约或排队号时，首页优先引导查看单据和叫号预测，减少重复操作。
- 运行中或出错时，首页按钮直接指向日志、诊断、停止或重新拿通行证。

### 诊断

- 本机诊断新增优先建议卡，先告诉用户最值得处理的一件事，再展示详细芯片和日志。
- 诊断建议覆盖代理残留、通行证不完整、证书未信任、代理链路异常、网络不可达、未选门店、未配置通知和引擎错误。

### GitHub 登录与 Turso 基准

- “验证连接”现在会同时验证 GitHub 会话和线上 Turso 排队基准接口；如果 Worker 到 Turso 失败，页面会直接显示具体错误。
- GitHub 登录回跳后，设置页会自动做一次 Turso 基准验证，不再只停留在“GitHub 已登录”。
- 图表在云端基准已连通但当前门店没有样本时，会说明“线上 Turso 基准已连接，但这家店暂时没有可用基准数据”，避免误以为没有调用数据库。
- 设置页新增“Turso 基准”状态芯片，区分未登录、未验证、已响应无样本和已验证样本数。

### 质量

- 新增前端体验锚点测试，保护首页指挥台和诊断建议不被后续重排误删。
- 新增云端基准链路测试，确保连接验证会实际访问基准接口，且登录回跳后会触发 Turso 验证。
