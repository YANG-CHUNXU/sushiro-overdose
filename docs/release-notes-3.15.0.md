# v3.15.0

## fix: MCP 助手配置卡用错样式类，重写为统一样式

MCP 卡片用了 `form-row` / `inp` / `row-wrap` 这几个**全项目都没定义**的 class，导致输入框、开关全是浏览器默认丑样式，和设置页其它卡片格格不入。

改为复用全站统一的 `.fg`（字段组）、`.check`（带勾选的胶囊）、`.bt`：标签在输入框上方、整齐换行；两个开关（启用 / 开机自启）与其它设置项一致；已配密钥时占位提示「已保存（保密不回显）」。

## feat: 凭证「真实取号」验证

本机录入的就是远程取号用的那套凭证；光验证查询/时段接口不等于取号能用。新增「验证凭证（取号测试）」按钮（通行证设置卡）：

- 找一家**正在开放线上取号**的门店，真实取号，然后**立即取消**——能取到=凭证仍生效，凭证类错误=该重新获取。
- **安全**：只取消本次验证刚取的号。若官方提示「你已有排队号」，说明凭证本就有效，此时**绝不取消**（那是你的真号），直接判有效；服务端临时错误判「无法确定」，不乱标失效。
- 已有活跃排队号时只读状态、跳过取号测试。
- 结果直接反馈，并联动凭证健康状态（有效→clear，失效→提醒重新获取）。
- 账号写操作，点击前有确认弹窗；取号/取消入口边界已在 `.specify/memory/constitution.md` 与架构守卫测试中登记。

## feat: 清理过去遗留的本地预约/排队记录

以前本机只清「跨天失效的排队号」，**过去的预约记录会一直挂着**不消失。

- 本地记录新增按日期过期：预约的 `QueueDate`/日期早于今天即判为遗留，自动从兜底列表清掉。
- 「我的单据」检测到本机遗留/补录记录时，给出**「清除本地遗留记录」**按钮（新端点 `/api/reservations/local/clear`）。只清本机缓存，不动寿司郎服务器上的真实单据，下次从官方重新同步。
