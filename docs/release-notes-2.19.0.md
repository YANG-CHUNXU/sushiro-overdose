# Sushiro Overdose 2.19.0 Release Notes

2.19.0 解决"认证悄悄失效、撞墙才知道"的问题，并把认证状态常驻到顶栏。

## 背景（为什么认证老失效）

工具复用的是手机/PC 微信抓包得到的**同一套令牌**，工具自身没有独立登录。寿司郎一个账号同一时间只认一个活跃会话——**在手机上用过小程序后，电脑这边的令牌就会被顶失效**；反过来用工具后，手机小程序也可能报"服务器出错"。这是复用令牌的先天限制，不是 bug。本版本不消除互斥（做不到），而是**令牌失效时第一时间提醒你重新捕获**。

## 主要变化

- **顶栏认证状态徽标**：常驻显示「只读模式 / 已认证 / 认证可能过期」，点击直达获取认证或我的单据。
- **认证过期提醒**：当读取/操作请求被官方判为认证失败时，首页顶部出现常驻红色横幅「认证可能已过期，点此重新捕获」，并通过已配置的通知渠道**推送一次**（不刷屏）。横幅文案点明"手机用过小程序后电脑认证会失效"，校正预期。
- **自动清除**：重新捕获/导入认证成功后，徽标回到「已认证」、横幅消失。
- 只看排队/数据图等只读功能不触发、不打扰；从未捕获过认证不会误报。

## 实现要点

- 纯被动检测：只看用户已触发的认证请求结果来判定，**不新增任何后台轮询**——因为任何工具主动请求都可能再把手机会话顶掉，定时探活只会添乱。
- 认证健康为进程内存状态，不落盘，重启不误报。
- 不触碰官方 mutation 与架构守卫边界；`go test ./...` 全绿（含架构守卫、前端契约守卫、认证健康状态机单测）。

## 不包含的变化

- 不消除"电脑用 vs 手机用"的会话互斥（先天限制）。
- 不做令牌自动续期（官方无此能力）。
- 不新增后台探活/轮询。
