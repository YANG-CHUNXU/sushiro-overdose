# Release notes: v3.0.0

3.0 是一次面向"没有技术背景、只想吃寿司的人"的整体体验改版（设计文档：docs/ux-redesign-v3.md），同时修复了普通用户拿不到线上排队基准的关键链路问题。

## 体验改版

- **三态首页**：游客态引导先选常用门店、马上看实时排队；就绪态显示任务卡与准备清单；有排队号/预约时单据卡置顶（当前叫号、预计几点叫到、一键设提醒）。
- **首屏即答案**：首页直接显示常用门店此刻"几桌在等 · 约几分钟 · 叫到几号"，点卡片直达实时排队；高级配置全部折叠，引擎状态空闲时隐藏。
- **通行证向导**：获取凭证改为五步状态机（选设备 → 抓一次 → 传到电脑 → 粘贴解析 → 验证），8 个字段实时点亮捕获进度，中途关闭可断点续做，完成有庆祝动效。第 1 步按平台分流：macOS 主推「PC 微信自动抓」，Windows 明确只有手机抓包 / 同 Wi-Fi 自动代理两条路。
- **只读功能零门槛出活**：「取号→几点吃」「想几点吃→几点取号」在没有历史样本时自动按当前实时等待粗估给出答案；常用门店的公开排队曲线默认在本机自动记录（无需通行证）。Routine 提醒维持"样本不足不提醒"。
- **常用门店记忆**：各页面选过的门店写入本机，刷新、重启不再重搜。
- **信息架构收敛**：自动蹲号并入「约未来」；历史洞察移入设置组；叫号预测页只保留答案卡与提醒设置，筛选器/KPI/曲线收进「图表与历史依据」（有数据时自动展开）。
- **多段到店提醒卡重做**：提醒节奏选项自带提前号数说明，自定义号码收进高级折叠，状态合并为一行。
- **健康胶囊**：右上角综合显示通行证 / 通知 / 预测数据三项前置条件，任何页面点开即可修复。
- **寿司家族与回转元素**：八只寿司宝宝吉祥物（三文鱼、金枪鱼、玉子烧、甜虾、鳗鱼、鱼子军舰、海苔卷、黄瓜卷），每页标题一只专属，页脚上方回转寿司传送带（尊重系统减少动效设置）。

## 关键修复

- **云端 Worker 无法解析 `libsql://` Turso 地址**：WHATWG URL 禁止把非特殊协议改写为特殊协议，原 `u.protocol="https:"` 静默失败导致 Worker 一律 500；改为解析前字符串替换并已重新部署。GitHub 登录后的全国排队基准（115 店 / 1.9 万条聚合）已验证可达。
- **云端响应 4MB 截断**：全国基准导出约 6.2MB，超出客户端读取上限导致"unexpected end of JSON input"；上限提至 64MB，超限给出明确错误，基准不可用警告附带错误原文。
- **应用重启后旧页面 CSRF 失败**：检测到 403 CSRF 自动提示并刷新自愈。
- **回转传送带循环错位与宽屏空白**：按视口铺满、两段严格对称、恒速滚动。

## 用户影响

- 双击即用：不登录、不抓包就能看全国排队、叫号预测和时间互推（实时粗估兜底）。
- 登录 GitHub 后，叫号预测与排队压力自动叠加线上全国基准。
- 老用户的本机数据、凭证与偏好不受影响；浏览器旧标签页过期会自动刷新。

## 验证

- 全量 go test 通过（含内嵌 HTML/JS 契约守卫与 node 语法门禁）。
- 隔离环境模拟普通用户：GitHub 会话→Worker→Turso 全链路取数 115 店成功。
- 真实环境（Turso 直连 + 本机采样）看板曲线 70 点正常。
- headless Chrome 截图冒烟：首页三态、欢迎屏、向导、叫号预测、传送带。
