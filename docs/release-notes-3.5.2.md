# Release notes: v3.5.2

v3.5.2 是一批用户反馈驱动的体验修复 + 数据准确性增强。

## 变化

### 整体走势图（修复容器溢出 + 去开店数）

- **图例区容器溢出**：图例项文案过长（"排队压力（灰带越满压力越大）"等）在窄屏溢出容器。精简图例文案（今日叫号 / 历史叫号 / 排队桌数趋势 / 排队压力 / 现在 / 我的号），并补响应式换行。
- **去掉「开店数」**：用户不需要。从趋势条和 tooltip 里移除"开店 N"。
- 走势图相关文案精简。

### 隐藏 Turso 字样

- 前端所有向用户展示的 "Turso" 字样统一改为「线上数据库」（设置页状态、图表来源说明、采样卡、GitHub 验证提示等）。`legend-turso-trend` 只是内部 CSS 类名，不对用户可见。

### 时间换算器标题随方向切换

- 几点取号 ⇄ 几点吃 切换 ⇄ 方向时，标题动态换：
  - 取号→吃：**「几点取号 → 几点能吃上」**
  - 吃→取号：**「想几点吃 → 该几点取号」**
- 副标题也随之变换，突出当前方向的重点。

### 现在去吃的自动取号：默认展开 + 无通行证提示

- qt 页「自动取号计划」默认展开（不再藏在折叠里）。
- 没配置寿司郎通行证时，状态区给醒目提示「自动取号需要通行证，点这里获取」，并附「拿通行证」按钮。

### 「我有号码」提醒输入框美化

- qdr/nr 系列输入框（提醒节奏、路上时间、备注、自定义号码）统一圆角、聚焦态红框、40px 高度，与全站组件对齐。

### 取号方案：恢复工作日/周末/节假日选择

- 「我有号码」页新增「参考曲线」下拉：全部日期 / 工作日 / 周末（含周五晚）/ 节假日。
- 后端原本就支持按 date_type 筛选历史曲线（周五 ≥16:30 算周末、节假日按 holidays.json），前端之前硬编码 all、没有入口，现在补回。
- ETA 预测用的就是选中日期类型的历史曲线。

### ETA 用官方等位封顶值钳制（数据准确性）

- 调研发现项目早就采集了官方 `waitTimeCap`（等位封顶，如 180 分钟）但没用。
- 现在用它做 ETA 软上限：模型算出的等待高位若明显超过官方封顶（多半是异常高峰/叫号停滞），钳到封顶值并提示「可能为异常高峰，建议到小程序确认」。避免给用户离谱预测。

### 屏幕自适应优化

- 中等宽度（≤900px）：控制条、表单行、换算行收敛换行，不再挤压。
- 窄屏（≤600px）：表单行竖向堆叠，标题字号、换算器头部自适应。

## 验证

- `gofmt` / `go build ./...` / `go vet ./...` / `go test ./...` 全绿（含新增 waitTimeCap 钳制测试）。
- CDP 真机回归：图例无溢出、开店数已移除、日期类型选择器存在、标题切换正常、移动端 390px 无横向溢出、零 JS 异常。

## 调研结论（寿司郎官方接口）

已用接口字段覆盖：门店信息、叫号队列、时段、预约/取号/取消、官方等待。
**已采集但还没充分利用**（留作后续）：`tablesCapacity`/`countersCapacity`（桌位容量，可归一化叫号速率做跨店对比）、`distance`（可做就近门店 + 出门时间）、`waitTimeCounter`（官方倒计时，可交叉校验）。本次先落地了 waitTimeCap 钳制（价值最高、最安全）。
