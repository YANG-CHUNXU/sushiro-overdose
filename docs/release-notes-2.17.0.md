# Sushiro Overdose 2.17.0 Release Notes

2.17.0 把「我有号码」的叫号曲线从一张数据图升级成可直接读出到店决策的图，并把「现在去吃」按使用频率重排。本次改动全部是只读前端，不改官方接口、后端数据结构或会执行操作的功能。

## 主要变化

- 叫号曲线新增「现在」竖线：按本地时间在 10:00–22:00 内标出当前位置，配「现在 HH:MM」标签。
- 叫号曲线新增「我的号」横线：输入号码后画出对应水平线，并高亮与一般叫号曲线的交点（与到店建议卡片一致）。
- 用阴影置信带替代原来的保守/偏快两条虚线，一眼读出不确定区间。
- 图例更新为「一般会叫到几号 / 保守-偏快区间 / 现在 / 我的号」。
- 叫号曲线与趋势曲线在窄屏（≤600px）自适应，不再强制横向滚动。
- 「现在去吃」重排：实时排队读数置顶；取号计划、全国门店基准采集、叫号提醒收进可折叠「高级」区。
- 「数据收集」从「现在去吃」移到「设置」分组，减少高频看排队时的干扰。

## 不包含的变化

- 不改官方 API 调用链路。
- 不改 `/api/queue/dashboard` 数据结构与接口契约（沿用 2.15 的 advisor 字段）。
- 不新增或改动取号/预约/取消等会执行操作的路径。
- 不引入图表库，曲线仍为手写零依赖 SVG。
- 趋势页暂不加现在/我的号标注。

## 验收点

- 在「我有号码」选门店、输入号码后，主图出现置信带、红色「现在」竖线、红色「我的号」横线和高亮交点，且与到店建议文字一致。
- 窄屏下主图与趋势图不出现横向滚动。
- 「现在去吃」首屏先看到实时排队，配置项折叠在「高级」内。
- 顶部不再有「本机收集」，「数据收集」出现在「设置」子导航并能正常加载。
