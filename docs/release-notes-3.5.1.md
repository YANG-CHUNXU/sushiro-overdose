# Release notes: v3.5.1

v3.5.1 把设置页从「7 个平铺折叠、像垃圾桶」重设计成**带分区的层级结构**，延续 v3.5.0 的动线主线。

## 变化

### 设置页分区重构

之前 7 个折叠平铺、没有分组、查找靠逐个点开。现在按 NN/G 的「分组卡片 + 清晰分区标题」组织成 4 个带编号的分区：

1. **凭证与认证** —— 寿司郎通行证 + GitHub 登录现在相邻（之前被通知隔开），统一为「认证类」。
2. **通知** —— 通知渠道单独成区。
3. **数据与预测** —— 预测准确度（采集）+ 历史洞察合并为「数据类」。
4. **排障** —— 诊断与维护 + 运行日志合并为「排障类」，从日常配置中分离。

每个分区有编号徽标 + 标题 + 右侧一句话说明，把「这里是什么、什么时候用」讲清楚。**危险操作**仍独立在最底部红框 Danger Zone（v3.5.0 已隔离）。

### 配套

- 顶部引导文案改为「按从上到下分区配置：凭证 → 通知 → 数据 → 排障」。
- 凭证/通知默认展开（v3.5.0 已做）。
- 通行证和 GitHub 相邻后，用户找认证类配置不再需要在通知折叠间跳。

## 验证

- `gofmt` / `go build ./...` / `go vet ./...` / `go test ./...` 全绿。
- CDP 真机回归：4 个分区标题正确渲染、通行证与 GitHub 相邻、所有关键 ID（cloudState/fold-sm/in/lo/safe 等）保留、Danger Zone 隔离、移动端无横向溢出、零 JS 异常。
