# Release notes: v3.1.0

3.1.0 一次性修复 v3.0.x UX 问题全集（21 项，依据 docs/ux-brief-v3.0.x.md），覆盖导航、首页层级、页面密度、设置整理与交互细节。

## 变化

### 导航与信息架构
- **「我的单据」加入顶部导航**：不再依赖散落的条件性入口；未拿通行证的访客进入会看到友好引导而不是接口报错。
- **浏览器后退键可用**：页面切换改用 `pushState` 并接管 `popstate`，按后退在页面间回退而不是直接退出应用。
- 首页任务卡标题「约未来 / 自动抢」统一为「约未来」，与顶部 Tab 一致。

### 首页
- 有活跃票据时 ticket-hero 始终是唯一英雄区；引擎运行/错误状态（含错误建议）归入侧栏引擎卡，不再双英雄区堆叠。
- 英雄区初始改骨架屏，不再闪「正在读取状态」；本机服务读不到时显示错误原文 + 重试按钮（之前会永久卡住）。
- 版本号 badge 拿到版本前隐藏，不再显示 "loading"。
- 首页实时排队：未选门店时给「选常用门店」引导、读取失败给重试；超过 3 家时显示「还有 N 家关注门店」溢出卡，不再静默截断/留白。

### 「我有号码」页降密度
- 主路径收敛为：实时排队压力（答案卡）→ 历史规律 · 到店建议 → 提醒；两个答案区加了差异化标题，不再写一样的 intro。
- 「本机持续采集」「时间换算」改为默认折叠，输入号码看答案无需滚动。
- 每日提醒状态占位改为中性「状态加载中…」，接口失败时显示错误原文 + 重试（之前静默残留过时文案）。

### 设置页
- 折叠顺序按使用频率调整：通行证 → **通知渠道** → GitHub → 预测 → 历史洞察 → 运行日志 → 安全与维护。
- 通知渠道一级只留「保存通知」「保存并测试全部」；4 个分渠道测试收进「更多操作」，并注明会先保存表单。
- 历史洞察表格不再显示裸 store_id（保留为 debug-only）。

### 「自动抢」与「现在去吃」
- 自动抢页重排：「现在就抢」「蹲还没放出的时段」两个操作模式各自成块；偏好拆为「人数与门店优先级」「日期与时段策略」两个折叠，告别一展开就是配置墙。
- 「取消排队号」从计划按钮行拆出，单独成行并加危险标记与说明（二次确认原本就有）。
- 实时排队加载态改骨架屏，失败态用统一错误盒（含原文与重试）。

### 交互细节与死链
- 所有弹层支持 Escape 关闭；健康面板、新手引导、门店选择器补点遮罩关闭。
- 修复 3 处 `go('sm')` 死链（历史洞察空态 ×2、每日提醒状态卡）：点「去预测准确度」会打开设置页对应折叠，不再静默跳回首页；顺带修复运行日志 SSE 实时行不自动滚动的死代码。

## 验证

- gofmt / go vet / go build / 全量 go test 通过（含 web_static 守卫：DOM id 引用、onclick 函数定义、id 唯一性、node --check JS 语法）。
- 修复后对 21 项逐条 grep 验证落位，无 `go('sm')`、`cp==='sm'/'lo'`、"loading" badge、旧加载占位残留。
