# Sushiro Overdose 2.15.0 Release Notes

## 重点

2.15.0 把排队数据看板从“看指标”升级成“做决策”：用户可以输入自己手里的排队号，直接看到大概几点叫到、建议几点前到店，以及当前哪个门店更值得去。

## 新增

- 数据看板新增“我的号”输入框：例如输入 `893` 后，系统会基于本机采集和线上公开基准估算最早叫到时间。
- `/api/queue/dashboard` 新增只读 `advisor` 字段，包含 `target_bucket`、`arrival_label`、`milestones` 和 `store_choices`，主项目可安全消费，不暴露数据库 token。
- 无号码时展示关键时间点：现在附近、1 小时后、2 小时后、晚高峰、收尾前大概叫到多少号。
- 目标号已过、目标号超出样本范围、没有曲线数据时都有明确中文提示。
- 门店建议会按营业状态、线上取号、前面桌数和等待分钟给出“最好去 / 可备选 / 偏挤 / 暂停”。

## 数据口径

- 叫号曲线固定展示 10:00-22:00。
- 支持 5、10、15、30 分钟颗粒度，默认 10 分钟。
- 同一天同一时间桶只取最后一条采样，避免高频采样放大某个时间点。
- 周五 16:30 到周日 22:00 按周末处理。
- 节假日和调休工作日继续读取 `~/.sushiro/holidays.json`，避免特殊日期污染普通工作日/周末基准。

## 安全边界

- 数据看板和 `advisor` 都是只读派生能力。
- 不会自动取号。
- 不会自动取消排队号或预约。
- 不会写入官方寿司郎状态。
- 不会暴露 Turso token；主项目只消费导出的 JSON/API 响应。
