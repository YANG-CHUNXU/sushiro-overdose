# v3.14.0

四个方向一起做：让通行证「失效→续期」无痛、让预测能自证可信、把核心移动场景从单向通知升级为自更新进度卡、顺手收敛设置页。

## feat: 通行证续期闭环——从「被动报错」到「主动续期」

凭证是所有预约系功能（抢预约、远程取号、读单据）的唯一闸门，过去失效后是个多步手动死胡同。本次补上「失效之后」那一步。

- **新增 `internal/app/auth_meta.go`**：和凭证本体（`config.json`）分开存的生命周期元数据——`captured_at`、捕获方式、历史寿命样本，算中位寿命。不动凭证文件格式与锁。
- **到期前柔性提醒**：凭证年龄达到「历史寿命中位数 ×0.8」时被动推一次「凭证快到期」（自带去重），不再等硬失效才打断。
- **失效通知可操作**：失效通知附带 `?recapture=1` 深链（点开直接拉起向导）+「沿用上次方式重抓」；失效跃迁时回填一段寿命样本喂给预测。
- **记住捕获方式**：PC 微信 / 手机抓包 / 手动导入三个真实捕获点记录方式，续期默认沿用。
- **前端**：横幅与设置页状态条显示「已用 N 天 / 快到期 / 沿用 X 方式」；启动读 `?recapture=1` 自动续期。

## feat: 预测精度自证回测——从「先验置信度」到「实测校准」

过去 `confidence` 只表示「数据多不多」，不代表「历史上准不准」。现在用实测对账闭环。

- **新增 `internal/app/queue_advisor_backtest.go`**：给出 ETA 时登记开放预测（首次为准），号被叫到时（`appendQueueObservation` 钩子）结算成 `eta_backtest.jsonl`，按店聚合 MAE / 系统性偏差。
- **反哺区间**：`buildQueueAdvisorEta` 用实测误差修正系统性偏早/偏晚并适度加宽区间，挂可信度话术。
- **新端点** `/api/queue/accuracy`；答案卡显示「这家店最近 N 次平均偏差 ±M 分钟」；设置页「预测准确度」新增各店实测误差表。
- 只统计「号确实被叫到」的客观时刻与预测之差，不掺用户是否按建议出发，避免行为污染。

## feat: 移动端自更新进度卡——同一目标号的多档提醒原地更新

「拿号后外出等待」是移动时刻，过去多档提醒会刷屏。

- **notify 包新增可选 `SessionNotifier` 接口** + `MultiNotifier.SendSession`，不支持的渠道自动回退普通推送。
- **Telegram 原地更新**：首条 `sendMessage`、后续 `editMessageText` 更新同一条消息；「not modified」视为成功；会话映射做成进程级，扛住每次重建 notifier。
- **接入** `queue_alerts.go`：同一店+目标号的多档提醒（还差 12→5→1 桌）共用会话键，更新同一张进度卡而非堆叠多条。

## refactor: 设置页收敛

- MCP 助手从「排障」错位中移出，单列「进阶 / 集成」分区。
- 状态总览的「通行证」项接入年龄/快到期；「预测数据」项接入「已用实测校准 N 家店」。
