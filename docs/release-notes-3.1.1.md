# Release notes: v3.1.1

3.1.1 修复 3.1.0 的 5 个残留/新引入问题（偏好折叠拆分的后遗症、入口语义）。

## 变化

- **「改抢号偏好」同时展开两个折叠**：3.1.0 把偏好拆成「人数与门店」「日期与时段」后，首页/自动抢页的入口只展开第一个折叠，最常改的时段策略要手动翻找；现在两个折叠一起展开，页内按钮与跨页入口走同一路径（`expandSnPrefs`）。
- **两个「保存偏好」按钮语义澄清**：改为「保存全部偏好」并注明人数、门店、时段是同一份偏好，在任意折叠保存都会一起保存。
- **「我的单据」Tab 随通行证状态显隐**：未拿通行证时该 Tab 对用户没有任何内容价值，隐藏（顶部回到 5 个 Tab）；拿到通行证后出现。页内入口（首页侧栏、ticket-hero 等）不受影响，访客经入口进入仍有引导兜底。
- **提醒卡「设置通知」按钮随状态改名**：已配置通知后显示「管理通知」（与健康面板「管理/去配置」惯例一致），不再对已配置用户假装没配置。
- **首页排队卡兜底语义明确化**：未显式选常用门店时，按通行证里你去过的门店兜底显示（只读路径保证有实时数据），并在卡片下方注明来源、附「选常用门店」按钮；不再让用户误以为这是自己选过的店。顺带删掉了语义混杂的 `homeWatchStores()`，显式选择与兜底两层在 `loadHomeLive` 内分开表达。

## 验证

- gofmt / go vet / go build / 全量 go test 通过（含 web_static 守卫与 node --check）。
