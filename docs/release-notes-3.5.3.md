# Release notes: v3.5.3

v3.5.3 落地官方接口调研里用户认可的「桌位容量」展示，并记录调研对其他几条的结论。

## 变化

### 实时排队卡展示桌位容量

- 之前 `tablesCapacity`/`countersCapacity`（桌位/吧台座席数）一直采集入库但零消费。
- 现在 qt 页「现在去吃」的实时排队卡底部展示「桌位 N / 吧台 M」，帮用户理解这家店的吞吐量（小店 6 桌 vs 旗舰店 80 桌，同样等位含义不同）。
- 没有容量数据时不显示，不破坏布局。
- 纯展示，不动预测算法（避免改 ETA 的回归风险）。

## 调研结论（寿司郎官方接口，记录存档）

用户对调研建议的反馈处理：
- ✅ **桌位容量**：落地（见上）。
- ❌ **官方倒计时 `waitTimeCounter`**：用户反馈不准，撤掉，不拿它做交叉校验。
- ⏸️ **距离/定位 `distance`**：需要位置权限，桌面工具体验差，暂不做。
- ⏸️ **小程序活动图片**：调研确认当前抓包数据里**没有任何营销/活动/图片接口的痕迹**（`StoreInfo` 无图片字段、12 个已知端点全是排队/取号/门店状态、discovery 从没跑过）。要确认得先开 API discovery 跑一遍小程序首页；且抓营销图偏产品核心（排队/取号），属 scope creep。**未做**，待产品明确要做"活动提醒"类功能再评估。
