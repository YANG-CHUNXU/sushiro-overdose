## ⚡ 即刻 /少数派 风格（带点技术人味，但仍然好懂）

**标题：我做了一个"寿司郎排队预测器"，核心功能就一句话**

吃完饭想分享个 side project。

起因特别朴素：每次取完号拿着小票，都不知道还要等多久、几点该走。小程序只告诉你"现在叫到几号"，但不告诉你"那你还要等多久"。

所以写了个工具，把这中间那一步算了：

- 拿到号 → 输进去 → 告诉你预计几点叫到 + 几点出发
- 根据最近的叫号速度、前面剩多少桌、历史规律一起算，不是拍脑袋
- 快叫到了自动提醒（支持飞书/TG/Bark/Server酱）
- 顺带能盯排队压力曲线、抢未来预约

技术栈是 Go + 单文件嵌入式 Web UI，零外部依赖，跑起来就是本地一个网页。所有数据在本地，凭证不上传。

**设计上有一个我比较得意的点**：「看排队预测」和「会动手的操作（取号/预约）」严格分开 —— 前者完全匿名、零登录、零风险；后者才需要"通行证"，而且每次执行前都会二次确认。我觉得工具类应用最该尊重用户的就是这个边界。

免费开源，欢迎试用和提建议：[链接]

## 🎯 V2EX / 技术社区版（更偏技术，但开头仍是场景）

**标题：[分享创造] 做了个寿司郎排队叫号预测器，解决"拿了号不知道还要等多久"**

各位 v 友，分享个最近做的 side project。

**解决的痛点**：寿司郎小程序只显示"当前叫到 X 号"，但拿到号的人真正想知道的是"我还要等多久 / 几点该出发"。这中间缺了一步估算。

**做了什么**：一个本地跑的 Go 程序（嵌入式 Web UI），把实时叫号速度、前面剩余桌数、历史规律、官方等待综合成一个 ETA 区间 + 出发时间建议。

几个我比较在意的设计取舍：
- **读写分离**：看排队/预测完全匿名零登录；只有取号/预约/取消这类"动手"操作才需要从手机小程序拿一次凭证（通行证），且执行前强制二次确认
- **凭证只存本地**（`~/.sushiro/`），不上传第三方；软件开源可审计
- **多源融合 ETA**：本机叫号速度 > 官方等待 > 历史分位数，会标注当前用的是哪个来源、可信度如何
- 支持 macOS/Win/Linux，自动抢预约、多渠道通知

开源地址：[链接]

## 💬 评论区常见问题预案（提前准备回复）

**Q：这算不算开挂/违规？会不会封号？**
A：用的全是寿司郎官方公开接口，和你用手机操作一模一样，不破解不外挂。只是帮你把"还要等多久"算出来 + 到点提醒你，不替你做任何违规操作。

**Q：准确吗？**
A：叫号速度是根据实时数据算的，越用越准（会记本机采样）。但毕竟是预测，给的是个区间（比如 38-62 分钟），不是精确到分。建议按偏早的时间出门。

**Q：要钱吗？**
A：完全免费，开源。

**Q：安卓/iOS 能用吗？**
A：现在是桌面版（Mac/Win/Linux），因为它需要在本地跑一个小服务。手机暂时没有原生 App。

**Q：安全吗？会偷我数据吗？**
A：开源的，代码全公开；你的凭证只存自己电脑上，不上传任何服务器；只看排队连登录都不用。

**Q：抢预约会不会抢到我不想要的时段？**
A：你可以指定门店、日期、时段范围，只抢你设置的。抢到前会确认。
