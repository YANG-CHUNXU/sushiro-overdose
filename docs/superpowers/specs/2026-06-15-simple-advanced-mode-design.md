# 简化版 / 进阶版模式设计

日期：2026-06-15

## 背景

SUSHIRO Overdose 的功能已经覆盖实时排队、叫号预测、未来预约、自动抢预约、远程取号、通知、采集、云端基准和本机诊断。问题不再是能力不足，而是普通用户打开 Web UI 后会被太多高阶功能和技术词分散注意力。

这次目标是上线一个默认的简化版，让用户先看到能直接帮助“今天吃上寿司郎”的必要看板；进阶版继续保留完整功能，给熟悉流程的用户使用。

现有系统适合做这个切分：

- Web UI 是 `internal/app/web_static.go` 中的单文件 `indexHTML`，不需要新增前端构建链。
- 顶部导航已经由 `NAV_GROUPS` / `PAGE_GROUP` / `go()` 驱动，适合按模式过滤。
- 偏好系统已经通过 `/api/preferences` 持久化，适合保存界面模式。
- 多数简化看板可复用现有只读 API，不需要第一阶段新增聚合端点。

## 目标

1. 默认进入简化版，首屏聚焦“今天怎么吃”和必要状态。
2. 简化版保留只读、低风险、常用的看板：今日状态、实时排队、我有号码、未来预约摘要、自动化概览。
3. 所有会执行操作、系统级维护、抓包细节、云端基准和深度分析默认收进进阶版。
4. 切换状态可持久化，刷新后仍保持用户选择。
5. 不引入 Node/npm，不新增发布复杂度，不破坏现有 release 流程。

## 非目标

- 不删除现有功能。
- 不重写整个 Web UI。
- 不在第一阶段新增 `/api/dashboard/summary` 聚合端点。
- 不改变预约、取号、取消、修复代理、卸载等 mutation 的后端权限边界。
- 不把抓包流程自动化为简化版主路径。

## 用户模式

### 简化版

简化版面向只想快速决策的普通用户。默认展示：

1. **今日状态**
   - 通行证状态：只显示未配置、可用、可能失效。
   - 通知状态：只显示是否配置。
   - 当前任务状态：空闲、运行中、错误。
   - 当前预约或排队号摘要：只显示必要时间和门店，详情点进完整页。

2. **实时排队**
   - 常用门店当前叫号、在等桌数、预估等待。
   - 允许选择门店和刷新。
   - 不默认暴露远程取号按钮。

3. **我有号码**
   - 输入当天排队号和门店，估算几点叫到、几点出发。
   - 保留实时压力、到店建议、走势大图和当次提醒。
   - 隐藏远程取号、每日自动取号提醒、采集配置等进阶块。

4. **未来预约摘要**
   - 首页保留“想约未来某天”的入口。
   - 简化版中点击时提示切换进阶版，而不是直接进入完整日历和自动抢预约。

5. **自动化概览**
   - 只显示采集、自动取号计划、蹲预约计划是否运行。
   - 不在简化版首页放启动、取消、修复、卸载等高风险操作。

简化版顶部导航建议只保留：

- 首页
- 现在去吃
- 我有号码
- 设置

设置页在简化版只展示：

- 界面模式切换
- 通行证状态和获取入口
- 通知配置
- 必要偏好

### 进阶版

进阶版保留完整功能：

- 约未来可约日历
- 自动抢预约
- 我的单据
- 远程取号和自动取号计划
- 每日取号提醒
- 预测准确度和持续采集
- 历史洞察
- GitHub / Turso 云端基准
- 手机凭证捕获和 UA 配置
- 运行日志
- 本机诊断、安全维护、修复代理、卸载、停止进程

进阶版顶部导航恢复完整分组：

- 首页
- 现在去吃
- 我有号码
- 约未来
- 我的单据
- 设置

## 交互设计

### 模式切换入口

在 header 的 `authPill` / `ver` 附近加入一个紧凑的分段切换：

- 简化
- 进阶

设置页顶部也放同一个模式控制，避免移动端 header 空间不足时找不到。

切换行为：

- 切到简化版：隐藏进阶导航和进阶区块；如果当前在进阶页面，回到首页并提示“已切回简化版”。
- 切到进阶版：恢复完整导航；不自动启动任何任务。
- 选择写入 `/api/preferences`，刷新后保持。

### URL hash 直达

简化版下用户可能直接访问 `#ca`、`#sn`、`#re`。

第一阶段采用保守行为：

- `go()` 入口拦截简化版不可见页面。
- 跳回首页或设置页。
- 显示提示：该功能在进阶版中，可切换后使用。

这样能避免用户误入含 mutation 的页面，也能避免隐藏 section 后定时器或初始化逻辑继续运行。

### 会执行操作

简化版必须隐藏或降级所有会执行操作入口：

- 远程取号
- 自动取号计划
- 取消排队号
- 自动抢预约
- 直接预约时段
- 修复代理
- 卸载
- 停止本应用相关进程

如果用户从简化版点击“想约未来”，显示切换进阶版的确认或说明，不直接提交任何动作。

## 数据流

第一阶段不新增后端聚合端点，前端复用现有接口：

- `/api/status`：全局状态、通行证、通知、引擎、采样状态。
- `/api/queue/live?store=ID`：首页和实时排队卡。
- `/api/queue/stores`：门店排队列表。
- `/api/queue/advisor`：我有号码的实时压力和叫号估算。
- `/api/queue/dashboard`：我有号码的走势大图和历史建议。
- `/api/reservations`：已有预约或排队号摘要。
- `/api/sampling`：自动化概览中的采集状态。
- `/api/sniper/plan`：进阶版蹲预约计划状态。
- `/api/queue/ticket/plan` 和 `/api/queue/ticket/routine`：进阶版自动取号和每日提醒状态。
- `/api/queue/alerts/status`：通知渠道和提醒状态，不直接读取 webhook/token。
- `/api/preferences`：保存 `ui_mode`。

隐私要求：

- 不在简化看板读取或展示 `/api/config` 中的 webhook/token。
- 不展示原始 `Authorization`、`User-Agent`、`Referer`、`wechat_id`。
- 预约号、排队号、ticket id 只展示必要摘要，详情由用户主动进入。

## 实现方案

推荐方案 A：前端模式切换 + 偏好持久化。

1. `UserPreferences` 增加 `ui_mode` 字段。
2. 默认值为 `simple`。
3. `NormalizePreferences` 将非法值回退为 `simple`。
4. `indexHTML` 增加模式切换控件。
5. JS 新增 `uiMode`、`setUIMode()`、`applyUIMode()`。
6. 简化版过滤顶部导航。
7. 简化版隐藏进阶 section 和进阶 details。
8. `go()` 拦截简化版不可见页面。
9. 首页“想约未来”卡在简化版中改为切换进阶版提示。
10. 增加静态测试保护关键 DOM、文案和函数。

第二阶段可选方案 B：新增 `/api/dashboard/summary`，集中做脱敏、缓存和外部接口节流。第一阶段不做，避免扩大上线面。

### 手工验收

1. 首次打开默认为简化版。
2. 顶部导航只显示首页、现在去吃、我有号码、设置。
3. 设置页切换进阶版后，约未来、我的单据等导航恢复。
4. 刷新后仍保持进阶版。
5. 切回简化版后，从 `#ca`、`#sn`、`#re` 直达会被拦截或提示。
6. 简化版不会展示远程取号、自动抢预约、取消排队号、修复代理、卸载等主操作。
7. 简化版实时排队和我有号码看板仍可正常使用。

## 风险和缓解

- **`web_static.go` 过大**：保持第一阶段改动集中，不拆前端构建链；用静态测试守住锚点。
- **hash 绕过**：在 `go()` 层拦截，而不只靠 CSS 隐藏。
- **mutation 入口残留**：给进阶块统一标识并增加静态测试。
- **移动端 header 挤压**：模式切换要紧凑，并在设置页提供第二入口。
- **GET 接口有外部请求**：简化首页复用已有缓存节流，不增加高频自动轮询。
- **老用户找不到功能**：简化版保留明显“切换进阶版”入口。

## 决策

采用方案 A 作为上线切片：默认简化版 + 进阶版切换 + 必要看板复用现有接口。暂不新增后端聚合端点，后续根据首页加载稳定性再评估 `/api/dashboard/summary`。
