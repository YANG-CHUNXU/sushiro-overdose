# v3.13.1

## feat: 凭证采集向导加 SVG 分步引导图

新人反馈"抓不全通行证"——根因是不知道要抓**两类请求**（门店查询 + 我的预约）。
本次给向导第 2 步加 SVG 流程图，第 3 步强化导出指引。

### 改动

- **第 2 步加 SVG 分步图**（`authCaptureFlowSVG`）：
  - ① 点门店 → 产生「查询请求」（查询auth · UA · referer）
  - ② 点我的预约 → 产生「预约请求」（预约auth · 微信ID · 手机号）
  - ③ 导出 cURL/请求头（两条都选）
  - ④ 粘贴到电脑 → 自动填好
  - 右侧警示"只抓一类就抓不全"
- **第 3 步强化导出指引**：
  - 明确"两条请求都要选"（对应图①②）
  - Stream 具体操作："点请求 → 右上角分享 → 复制 cURL"
  - 两条内容粘一起即可（parser 会合并解析）

### 为什么

之前向导只有文字步骤，新人容易只抓一类请求导致字段不全。SVG 图一眼看清"抓什么、抓几次"，
第 3 步明确"两条都选 + Stream 怎么导出"，降低操作出错。

> 仅改 `web_static.go`（前端），桌面端 Go 逻辑无改动。
