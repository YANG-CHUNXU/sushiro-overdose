# Release notes: v3.0.1

3.0.1 是 3.0 的"化繁为简"修订：删掉重复与低频功能、合并同类入口，主导航收敛为 首页 · 现在去吃 · 我有号码 · 约未来 · 设置。

## 精简与合并

- **「自动抢」统一入口**：自动抢预约（已放出时段按偏好开抢）与自动蹲号（蹲未放出时段）合并为一页，首页按钮直达。
- **「我有号码」一页搞定**：取号⇄就餐时间互推合并为页内换算卡（删两个子页）；多段出发提醒与每日取号提醒（原 Routine）合并为提醒卡双 tab，全站只有一个提醒入口。
- **设置页重排**：顶部改为四条状态总览（通行证 / GitHub 线上基准 / 通知 / 预测数据，每条一个动作按钮），删除重复出现三遍的动作按钮和深色横幅；折叠默认收起、加悬停反馈。「预约与取号偏好」整卡迁入「自动抢」页（人数 / 门店优先级 / 时段就在用它的地方），设置页只剩账号与系统。
- **收编与隐藏**：预测准确度、历史洞察、运行日志收进设置页折叠（展开才加载）；「我的单据」从主导航隐藏（官方列表接口不稳定），从首页单据卡、抢号成功页、偏好卡进入。
- **真删**：接口调试、移动端 UA 采集、手动导入凭证调试区（向导第 4 步承接）、「现在去吃」的历史趋势分析图表（与叫号预测页重复）。前端净删约 50 个函数与 6 个页面区块。

## 修复

- **提醒状态接口 panic**：给一家还没有本机观测记录的门店设置提醒后，`/api/queue/alerts/status` 对空观测序列取下标崩溃（新用户首次设提醒即触发）；已修复并回归。

## 验证

- 全量 go test 通过（含内嵌 HTML/JS 契约守卫与 node 语法门禁）。
- 游客链路实测：看排队、单店实时、时间换算实时兜底、提醒生成与状态、通行证解析。
- 真实环境实测：本机叫号曲线 70 点、单店压力线上补全 31 点、GitHub 会话与日历接口正常。
